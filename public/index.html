📋 Overview

This tool calculates credit prices for crafted items in the game Eco using calorie-based economics. You build a recipe database, configure your character's skills and crafting table upgrades, then look up any item to see its fully-loaded cost — including all nested sub-ingredients — converted to credits at your server's calorie rate.

All data (recipes, skills, tables, tags) is stored in your browser's localStorage. Use Export to back up your database and Import to restore or share it.

⚡ Quick Start

1. Configure skills
Click ⚙ Skills & Tables and set each skill to your character's in-game star level.
2. Add your recipes
Click + New in the Recipe Database, or click ↺ Seed Raws to pre-populate common raw materials. You can also Import a JSON export.
3. Set the calorie rate
Enter your server's Credits per Calorie in the Calculator panel (e.g. 1.6 if 1000 cal = 1600 credits).
4. Price an item
Click any item in the list — the Calculator instantly shows its credit price and full ingredient cost tree.

📦 Recipe Database (Left Panel)

Search
Filter the list by typing any part of an item name.
Group By
Sort the list alphabetically (A–Z), or group by Type (raw/crafted), Skill, or Crafting Table.
+ New
Create a new blank item and open it in the editor.
↺ Seed Raws
Adds a built-in list of common raw materials (ores, wood, stone, etc.) to your database. Existing items are not overwritten. Set each raw material's Market Buy Price or Labor Calories to reflect your server's costs.
Click an item
Opens the item in the editor and immediately runs the calculator for that item.
Missing recipe (red)
If an ingredient has no recipe in the database, it appears red in the cost tree. Click it to create a new recipe pre-filled with that name.

✏️ Item Editor (Middle Panel)

Fields apply to the item as a whole unless noted as per-recipe.

Item Name
The item's display name. This is what other recipes reference when listing this as an ingredient.
Market Buy Price
Fixed cost in credits per unit. When set, the calorie calculation is bypassed — the item is treated as a direct credit purchase. Leave blank to calculate cost from the recipe tree.
Item Tags
Comma-separated tag names this item belongs to (e.g. Ore, Metal). Tags are virtual ingredient categories — another recipe can list Ore as an ingredient and the cost will be the average of all items tagged as Ore. See Item Tags.
Multiple Recipes
An item can have multiple recipe variants (e.g. two wood types). Use the tabs above the form to switch between them. Click ★ Set Default on the tab you want used in cost calculations.
Recipe Label
(per-recipe) A name for this variant shown in the tab and in the calculator dropdown.
Skill
(per-recipe) The skill required to craft this item. Must match a name in your Skills settings (case-insensitive). The skill's level determines the labor calorie multiplier.
Crafting Table
(per-recipe) The table used to craft this item. Must match a name in your Tables settings. The table's upgrade module reduces ingredient quantities and optionally labor.
Labor Points
(per-recipe) Labor points consumed per craft batch (as shown in the Eco recipe UI). Treated as 1:1 with calories. For raw/gathered materials, set this to the gathering cost. Scaled by skill level and (optionally) table upgrade.
Qty Produced per Craft
(per-recipe) How many units one craft produces. The calculator divides total cost by this number to get the per-unit price.
Min. Skill Level
(per-recipe) Informational — the minimum skill level required to unlock this recipe in-game. Recipes below this threshold are flagged in the cost tree.
Profit Override %
(per-recipe) Overrides the global Profit Margin for this specific item. Leave blank to use the global setting.
Ingredients
Items consumed per craft. Each has a name, quantity, and a Red. (Reducible) checkbox. Reducible ingredients are scaled by the table upgrade, lavish workspace, and craft modifier. Uncheck for fixed costs that are never reduced.
Byproducts
Items produced as a side effect. Their calorie value is subtracted from the total cost — if a byproduct has its own recipe, its full tree value is credited back.

🧮 Calculator (Right Panel)

Credits per Calorie
Your server's calorie-to-credit conversion rate. Example: if 1000 calories costs 1600 credits, enter 1.6. This is the most important setting — all prices scale linearly with this value.
Currency Symbol
Cosmetic label shown next to prices (e.g. $, ¢, G).
Profit Margin %
Percentage markup added on top of the base cost. A value of 20 means the suggested price is 20% above cost. Per-item overrides are available in the editor.
Craft Modifier
Global multiplier applied to all reducible ingredient quantities. Use this for server-wide craft rate modifications (e.g. 0.5 if your server halves all resource costs). Does not affect non-reducible ingredients or labor.
Table mod affects labor
When checked, the crafting table upgrade module reduces labor calories in addition to ingredient quantities. Uncheck to match calculators that only apply upgrades to ingredients.
Search & Qty
Type an item name and press Enter to price it. Use the Qty field to price a batch. If an item has multiple recipe variants, a dropdown appears to select which one to use.
Copy button
Copies the final calculated price to your clipboard.
Cost tree
Expands the full ingredient breakdown. Each node shows the calorie cost and credit equivalent. Labor is shown in teal, byproducts in green (as savings), missing recipes in red. Click a missing recipe to create it.

⚙️ Skills & Crafting Tables

Skill Level (0–7)
Matches the star rating shown in-game. Level 0 = not learned (no labor reduction). Level 7 = fully specialised (50% labor). The multipliers are: 0→100% 1→80% 2→75% 3→70% 4→65% 5→60% 6→55% 7→50%
Lavish Workspace
Check if your character has the Lavish Workspace talent active for that skill. Applies an additional −5% to all reducible ingredient quantities (not supported by all skills, e.g. Mining does not support it).
Table Upgrade
Assign an upgrade module to each crafting table. Options range from No Upgrade (×1.0) through Basic I–IV, Advanced I–IV, up to Specialized (×0.50). The modifier reduces reducible ingredient quantities and, if "Table mod affects labor" is checked, also reduces labor calories.
Reset to Vanilla Defaults
Resets all skills to the built-in list at level 1 and all tables to No Upgrade. Your recipe database and item tags are not affected.

🏷️ Item Tags

Some Eco recipes accept any item from a category (e.g. "any Ore"). Item Tags model this — a tag is a named group of items that can be used as an ingredient name in recipes.

Creating tags
Open an item in the editor and type a tag name in the Item Tags field. The tag is created automatically and this item is added as a member. Multiple tags can be comma-separated.
Using tags as ingredients
In another item's recipe, type the tag name in the ingredient list (it appears with a [tag] label in the autocomplete). The cost is the average of all member items' calculated costs.
Tag market price override
In ⚙ Skills & Tables → Item Tags, you can set a market price override for a tag (in credits). When set, this overrides the member average — useful if you buy all ores at a fixed market price regardless of type.
Tag display
Tags appear in purple in the cost tree with a badge showing "tag · avg of N members".

💾 Import / Export

Export
Opens a panel with your full database as JSON — items, skills, tables, and item tags. Copy and save this text to back up or share your setup.
Import
Paste a JSON export to load a database. This replaces your current data. Use the Schema tab for the expected format.
JSON format: An array of item objects. Each item needs at minimum a name and a recipes array.

[ { "name": "Wooden Chair", "recipes": [{ "label": "Default", "skill": "Carpentry", "craftingTable": "Workbench", "laborPoints": 200, "outputQty": 1, "ingredients": [{"name": "Wood Board", "qty": 4}], "byproducts": [] }] } ]

Raw materials have no ingredients and use laborPoints for the gathering cost, or set a marketPrice on the item to use a fixed credit cost.

📐 Cost Calculation Formula

Costs are calculated recursively — each ingredient's full tree is resolved before the parent item is totalled.

reducible_mult = table_upgrade × lavish(0.95 if on) × craft_modifier
labor_mult     = skill_level_modifier × table_upgrade*
* only if "Table mod affects labor" is checked

ingredient_cal = Σ ( qty × reducible_mult × ingredient_cost_cal ) ← reducible
               + Σ ( qty × ingredient_cost_cal ) ← non-reducible
labor_cal      = laborPoints × labor_mult × batches_needed
byproduct_cal  = Σ ( byproduct_qty × byproduct_cost_cal )

net_cal        = ( labor_cal + ingredient_cal − byproduct_cal ) / output_qty
price         = net_cal × credits_per_calorie × ( 1 + profit% / 100 )
Market-priced items bypass the formula — their cost is market_price × credits_per_calorie⁻¹ calories (i.e. the credit cost converted back to calories at the current rate).
Tag ingredients cost the average of all member items' net_cal values, or the tag's market price override if set.
